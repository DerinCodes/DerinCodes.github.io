Maze Generation Algorithms

This page explores four maze generation algorithms—Depth-First Search, Prim’s Algorithm, Kruskal’s Algorithm, and Recursive Division—highlighting their pros and cons, code snippets, and an example maze image.

Depth-First Search (DFS)

DFS is a backtracking algorithm that creates a maze by starting at a cell and exploring as far as possible along each branch before backtracking.

Pros: Fast, simple, generates mazes with long, winding passages.
Cons: Can result in very few junctions or dead ends.

void DFS(Vector2Int cell) {
  visited[cell.x, cell.y] = true;
  foreach (Vector2Int dir in directions) {
    if (IsValidMove(cell + dir)) DFS(cell + dir);
  }
}

Prim’s Algorithm

Prim's Algorithm grows a maze by randomly connecting frontier cells to already visited cells in the maze.

Pros: Creates balanced mazes with a mix of short and long paths.
Cons: Can generate short, cul-de-sac-filled mazes that are more predictable.

void GenerateMaze() {
  MarkInMaze(start);
  while (!frontier.empty()) {
    Vector2Int cell = GetRandomFrontierCell();
    ConnectToMaze(cell);
  }
}

Kruskal’s Algorithm

Kruskal’s algorithm connects random walls between unconnected cells to build a spanning tree of paths across the maze.

Pros: Generates a fully connected maze with diverse path lengths.
Cons: Can be computationally expensive for large mazes.

void KruskalsMaze() {
  foreach (Wall wall in randomWalls) {
    if (ConnectsDisjointSets(wall)) RemoveWall(wall);
  }
}

Recursive Division

Recursive Division divides the maze area with walls and randomly placed openings, creating a maze with large sections.

Pros: Creates visually distinct mazes with large open sections.
Cons: Can result in overly open spaces or large isolated sections.

void Divide(int x, int y, int width, int height) {
  if (width <= 2 || height <= 2) return;
  AddWall(x, y, width, height);
  Divide(newArea1); Divide(newArea2);
}
